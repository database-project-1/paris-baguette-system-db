## 0. 시작하기 전에 (필수 사전 준비)

### 0-1. DB 파일 준비

`파리바게트.db`는 용량 문제로 GitHub에 올리지 않았습니다. 아래 Google Drive 링크에서 다운로드한 뒤 `schema/` 폴더에 넣어주세요.

> 📁 **Google Drive 링크:** `[🔗](https://drive.google.com/drive/folders/1Fy7RQH6XqRe04M522_9jzh_Ounxf_wdx?dmr=1&ec=wgc-drive-%5Bmodule%5D-goto)`

DB 파일이 없거나 처음부터 새로 만들고 싶다면 1번 순서대로 직접 생성할 수도 있습니다.

### 0-2. 실행 환경

- Python 3.9 이상
- 표준 라이브러리(`sqlite3`, `uuid`, `pathlib`, `datetime`) 외 추가 설치 필요:
  ```bash
  pip install faker
  ```
- 모든 인터페이스는 **Jupyter Notebook(.ipynb)** 형식이며, Jupyter Notebook / JupyterLab / VS Code의 Jupyter 확장 등에서 실행 가능합니다.

### 0-3. 실행 순서 (필독)

DB를 처음부터 만드는 경우 반드시 아래 순서를 지켜야 합니다. 순서가 바뀌면 "테이블이 없다" 또는 "삽입 순서" 오류가 발생합니다.

```
1) schema/dbschema.ipynb   실행  →  테이블(스키마) 생성
2) data/data.ipynb         실행  →  더미 데이터 삽입
3) interfaces/ 안의 원하는 인터페이스 실행
```

이미 다운로드한 `파리바게트.db`를 사용한다면 1), 2)는 생략하고 바로 3)부터 진행하면 됩니다.

## 1. DB 생성 (처음 한 번만)

### `schema/dbschema.ipynb`
- `schema/schema.sql`에 정의된 DDL을 읽어서 SQLite DB(`파리바게트.db`)에 31개 테이블을 생성합니다.
- 기존 DB가 있다면 전체 DROP 후 재생성하므로 **주의해서 실행**하세요.
- 실행 후 "생성된 테이블 31개: [...]" 메시지가 뜨면 성공입니다.

### `data/data.ipynb`
- `Faker(ko_KR)` 라이브러리로 지점, 상품, 고객, 회원, 직원, 판매 등 더미 데이터를 생성해 DB에 삽입합니다.
- **반드시 dbschema.ipynb 실행 후에 실행해야 합니다.**

## 2. 인터페이스 실행 방법

> ⚠️ **공통 주의사항:** 아래 7개 인터페이스는 모두 `interfaces/` 폴더 안에서 실행하는 것을 기준으로 작성되었습니다. (DB 경로를 상대경로 `../schema/파리바게트.db`로 찾거나, 프로젝트 루트를 자동 탐색합니다.)

각 노트북은 단일 셀로 구성되어 있습니다. **Jupyter에서 해당 노트북을 열고 셀을 실행(Run)** 하면 그 아래 출력창에 메뉴가 표시되고, 입력창에 번호/값을 입력하며 진행하는 방식입니다.

### ① 소비자 CLI — `consumer_cli.ipynb`
| 항목 | 내용 |
|---|---|
| 사용자 | 회원 / 비회원 소비자 |
| 기능 | 로그인·비회원 진입, 매장 선택, 상품 검색(이름/카테고리/브랜드), 구매, 구매이력 조회, **환불(당일 구매건만)** |
| 실행 | 셀 실행 → `1. 회원 로그인` 또는 `2. 비회원으로 구매` 선택 |

### ② 매장 직원 POS — `staff_Pos.ipynb`
| 항목 | 내용 |
|----|---|
| 사용자 | 매장 직원 / 지점장 |
| 기능 | 직원 로그인(로그인 화면에 지점장 ID 목록 표시), POS 결제, 매장 재고 조회(최소재고기준 미달 ★표시), 수동 발주 요청, 발주 상태 조회, **결제내역 조회·환불(지점장 전용, 당일 구매건만)** |
| 실행 | 셀 실행 → 직원 ID 입력 후 로그인 |
| 비고 | 지점별로 가장 먼저 등록된 직원이 `지점장`, 나머지는 `직원`으로 구분되어 있으며 환불 권한은 지점장에게만 있습니다. |

### ③ 회원가입/동의관리 — `member_registration.ipynb`
| 항목 | 내용 |
|---|---|
| 사용자 | 신규 / 기존 회원 |
| 기능 | 신규 회원가입(아이디 중복확인, 비밀번호 유효성 검사), 동의내역 조회, 동의 변경/철회(이력 보존), 회원 탈퇴 |
| 실행 | 셀 실행 → 메뉴 번호 선택 |

### ④ 공급업체 공급 처리 — `supplier_cli.ipynb`
| 항목 | 내용 |
|---|---|
| 사용자  | 공급업체 담당자 |
| 기능 | 업체정보 조회, 미처리 발주 목록/상세 조회, 공급 완료 처리(발주 상태 대기→완료 갱신), 공급 이력 조회, 재고 반영 확인 |
| 실행 | 셀 실행 → DB 자동 탐색 후 메뉴 진행 |

### ⑤ 자동 재주문 발주 프로그램 — `auto_reorder.ipynb`
| 항목 | 내용 |
|---|---|
| 사용자 | 스케줄러(자동 실행) |
| 기능 | 전 매장 재고수량 < 최소재고기준 스캔 → 공급업체별 그룹화 발주 → 중복 발주 방지 → 발주/발주상세 자동 생성 → 실행 로그 기록 |
| 실행 옵션 | `--db [경로]` DB 직접 지정 / `--execute` 미리보기 없이 실제 발주 생성 / `--no-log` 로그 저장 생략 |
| 로그 위치 | `interfaces/logs/` |

### ⑥ DBA 콘솔 — `dba_console.ipynb`
| 항목 | 내용 |
|---- |---|
| 사용자  | 데이터베이스 관리자 |
| 기능 | 테이블 목록/구조 조회, SQL 또는 도트 명령 실행, 트랜잭션 제어(BEGIN/COMMIT/ROLLBACK), 인덱스 목록 조회, 제약조건/무결성 점검, 테이블별 행 개수 확인 |
| 실행 | 셀 실행 → DB 자동 탐색 후 메뉴 진행 |

### ⑦ OLAP 분석 대시보드 — `olap_dashboard.ipynb`
| 항목 | 내용 |
|---- |---|
| 사용자  | 본사 운영팀 |
| 기능 | 매장별/지역별 베스트셀러 TOP N, 판매 실적 상위 매장, 브랜드별 매출 비교, 연관 상품 분석, 카테고리별 매출 비중, 재고-판매수량 비교, 월별 매출 추이 |
| 실행 | 셀 실행 → DB 자동 탐색 후 메뉴 번호/매장명/기간 등 입력 |

## 4. 자주 발생하는 문제

| 증상 | 원인 / 해결 |
|---|---|
| `FileNotFoundError: DB 파일을 찾을 수 없습니다` | `interfaces/` 폴더에서 노트북을 실행 중인지 확인. `schema/파리바게트.db`가 실제로 존재하는지 확인 |
| 테이블이 없다는 오류 | `dbschema.ipynb`를 먼저 실행하지 않은 경우. 순서 0-3 참고 |
| 데이터 삽입 시 FK 오류 | `dbschema.ipynb` → `data.ipynb` 순서를 지키지 않은 경우 |
| 회원 데이터 컬럼이 뒤섞여 보임 | 이전 버전의 `data.ipynb` 오류였으며 현재 버전에서는 수정 완료 |
